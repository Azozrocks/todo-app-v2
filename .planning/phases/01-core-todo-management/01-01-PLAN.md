## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell/argv -> CLI | User-supplied command name and item text enter the process untrusted |
| CLI -> filesystem | Reads/writes todos.json in the current working directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | lib/storage.js readTodos() | medium | mitigate | JSON.parse wrapped in try/catch; non-array or unparseable content -> clear stderr message + exit 1 (no crash, no partial trust) |
| T-01-02 | Denial of Service | lib/storage.js reading a huge/malformed todos.json | low | accept | Local single-user tool operating on a file the user owns; no untrusted network input |
| T-01-03 | Tampering | Path resolution for todos.json | low | mitigate | Path is always path.join(process.cwd(), 'todos.json'); never derived from user argv, so no path-traversal surface from the item text |
| T-01-04 | Elevation of Privilege | add handler writing arbitrary content | low | accept | Content is a plain string stored under a `text` field; no code execution / eval path |
| T-01-SC | Tampering | npm/dependency supply chain | low | accept | Zero npm dependencies (Node built-ins only); no install step in this plan — nothing to audit |

No HIGH-severity threats (block-on threshold is `high`); nothing blocks execution.
